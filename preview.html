Tech stack

FrontendVanilla HTML + CSS + JavaScript (no framework, no build step)
IconsInline SVG, designed to look sharp on Retina & readable at 36–72 px
PWA shellmanifest.json + apple-mobile-web-app-capable meta tags for full-screen home-screen launch
Data fetchfetch() directly from browser — Open-Meteo is CORS-enabled, no proxy needed
RefreshsetInterval every 15 min + on visibility change (when you re-open the app)
HostingGitHub Pages (free). Alt: Cloudflare Pages, Netlify, or just file:// locally
BackendNone. 100% client-side.

Data source — Open-Meteo HRRR

Open-Meteo offers NOAA's HRRR model directly. The full request we'd issue:

https://api.open-meteo.com/v1/forecast
  ?latitude=40.7644&longitude=-73.9594          ← zip 10065 centroid
  &models=hrrr
  &hourly=temperature_2m,cloud_cover,precipitation,wind_speed_10m,wind_direction_10m
  &temperature_unit=celsius
  &wind_speed_unit=mph
  &precipitation_unit=mm
  &timezone=America/New_York
  &forecast_hours=48                              ← HRRR's full free window
  • Free, no API key, unlimited reasonable use for personal apps
  • HRRR runs hourly; Open-Meteo serves the latest run automatically
  • Returns 48 hourly forecast steps (HRRR's max horizon)

How "full-screen on iPhone" works

  1. Open the GitHub Pages URL in Safari on the iPhone
  2. Share → "Add to Home Screen" → tap the new icon
  3. App opens with no browser chrome thanks to apple-mobile-web-app-capable + display: standalone in manifest.json

2. Sky Condition Icons (16 types)

Combinations of cloud cover (Open-Meteo cloud_cover %) and rain intensity (precipitation mm/hr). Day/night switches via Open-Meteo's is_day.

Cloud bands: Clear 0–12% · Few 12–37% · Partly 37–62% · Mostly 62–87% · Overcast 87–100%
Rain bands (mm/hr): Dry 0 · Light 0.1–2.5 · Moderate 2.5–7.6 · Heavy >7.6 (NWS scale)

3. Wind Direction Markers

Triangle sits on the circle perimeter at the direction the wind is going to, then rotated 29° counter-clockwise so that the iconic Manhattan grid (29° E of true N) reads "up = uptown / right = crosstown east."

Formula used: display_angle = ((wind_from_deg + 180) − 29 + 360) mod 360
Sanity check: wind from W (270°) → going to 90° → minus 29° → 61° on circle ✓ (matches your example)

Examples — pure compass (no Manhattan adjustment) vs. Manhattan-adjusted (what the app will use)
